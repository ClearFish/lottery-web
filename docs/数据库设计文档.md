# 数据库设计文档

## 数据库概述

本系统采用 MySQL 作为主数据库，Redis 作为缓存数据库。MySQL 负责数据持久化存储，Redis 负责会话管理、缓存和实时数据。

## 表结构设计

### 1. 用户表 (users)

```sql
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  username VARCHAR(50) UNIQUE NOT NULL COMMENT '用户名',
  phone VARCHAR(20) UNIQUE NOT NULL COMMENT '手机号',
  password VARCHAR(255) NOT NULL COMMENT '密码(加密)',
  invite_code VARCHAR(20) UNIQUE NOT NULL COMMENT '邀请码',
  parent_id INT COMMENT '上级用户ID',
  balance DECIMAL(10,2) DEFAULT 0.00 COMMENT '账户余额',
  total_earnings DECIMAL(10,2) DEFAULT 0.00 COMMENT '总收益',
  status ENUM('active', 'inactive', 'banned') DEFAULT 'active' COMMENT '账户状态',
  last_login_at TIMESTAMP NULL COMMENT '最后登录时间',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  
  INDEX idx_phone (phone),
  INDEX idx_invite_code (invite_code),
  INDEX idx_parent_id (parent_id),
  INDEX idx_status (status),
  FOREIGN KEY (parent_id) REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 2. 任务表 (tasks)

```sql
CREATE TABLE tasks (
  id INT PRIMARY KEY AUTO_INCREMENT COMMENT '任务ID',
  name VARCHAR(200) NOT NULL COMMENT '任务名称',
  category ENUM('douyin', 'kuaishou', 'sms', 'xiaohongshu', 'toutiao') NOT NULL COMMENT '任务分类',
  reward DECIMAL(8,2) NOT NULL COMMENT '奖励金额',
  total_count INT NOT NULL DEFAULT 0 COMMENT '总数量',
  remaining_count INT NOT NULL DEFAULT 0 COMMENT '剩余数量',
  keywords TEXT COMMENT '关键词(JSON格式)',
  description TEXT NOT NULL COMMENT '任务描述',
  requirements TEXT NOT NULL COMMENT '任务要求',
  video_materials JSON COMMENT '视频素材(JSON格式)',
  image_materials JSON COMMENT '图片素材(JSON格式)',
  difficulty ENUM('easy', 'medium', 'hard') DEFAULT 'easy' COMMENT '难度等级',
  estimated_time INT DEFAULT 0 COMMENT '预计完成时间(分钟)',
  status ENUM('active', 'inactive', 'completed') DEFAULT 'active' COMMENT '任务状态',
  sort_order INT DEFAULT 0 COMMENT '排序权重',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  
  INDEX idx_category (category),
  INDEX idx_status (status),
  INDEX idx_sort_order (sort_order),
  INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='任务表';
```

### 3. 用户任务表 (user_tasks)

```sql
CREATE TABLE user_tasks (
  id INT PRIMARY KEY AUTO_INCREMENT COMMENT '用户任务ID',
  user_id INT NOT NULL COMMENT '用户ID',
  task_id INT NOT NULL COMMENT '任务ID',
  status ENUM('pending', 'in_progress', 'completed', 'abandoned', 'rejected') DEFAULT 'pending' COMMENT '任务状态',
  claimed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '领取时间',
  started_at TIMESTAMP NULL COMMENT '开始时间',
  completed_at TIMESTAMP NULL COMMENT '完成时间',
  reward_amount DECIMAL(8,2) DEFAULT 0.00 COMMENT '实际奖励金额',
  submission_data JSON COMMENT '提交数据(JSON格式)',
  admin_notes TEXT COMMENT '管理员备注',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  
  INDEX idx_user_id (user_id),
  INDEX idx_task_id (task_id),
  INDEX idx_status (status),
  INDEX idx_claimed_at (claimed_at),
  UNIQUE KEY uk_user_task (user_id, task_id),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户任务表';
```

### 4. 提现记录表 (withdrawals)

```sql
CREATE TABLE withdrawals (
  id INT PRIMARY KEY AUTO_INCREMENT COMMENT '提现ID',
  user_id INT NOT NULL COMMENT '用户ID',
  amount DECIMAL(10,2) NOT NULL COMMENT '提现金额',
  method ENUM('alipay', 'wechat', 'bank') NOT NULL COMMENT '提现方式',
  account VARCHAR(100) NOT NULL COMMENT '提现账户',
  real_name VARCHAR(50) NOT NULL COMMENT '真实姓名',
  status ENUM('pending', 'approved', 'rejected', 'completed') DEFAULT 'pending' COMMENT '提现状态',
  admin_notes TEXT COMMENT '管理员备注',
  processed_at TIMESTAMP NULL COMMENT '处理时间',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '申请时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  
  INDEX idx_user_id (user_id),
  INDEX idx_status (status),
  INDEX idx_created_at (created_at),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='提现记录表';
```

### 5. 收益记录表 (earnings)

```sql
CREATE TABLE earnings (
  id INT PRIMARY KEY AUTO_INCREMENT COMMENT '收益ID',
  user_id INT NOT NULL COMMENT '用户ID',
  type ENUM('task', 'promotion', 'bonus', 'withdrawal') NOT NULL COMMENT '收益类型',
  amount DECIMAL(8,2) NOT NULL COMMENT '金额',
  source_id INT COMMENT '来源ID(任务ID或用户ID)',
  description VARCHAR(200) COMMENT '描述',
  balance_before DECIMAL(10,2) NOT NULL COMMENT '操作前余额',
  balance_after DECIMAL(10,2) NOT NULL COMMENT '操作后余额',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  
  INDEX idx_user_id (user_id),
  INDEX idx_type (type),
  INDEX idx_created_at (created_at),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='收益记录表';
```

### 6. 团队关系表 (team_relations)

```sql
CREATE TABLE team_relations (
  id INT PRIMARY KEY AUTO_INCREMENT COMMENT '关系ID',
  parent_id INT NOT NULL COMMENT '上级用户ID',
  child_id INT NOT NULL COMMENT '下级用户ID',
  level TINYINT NOT NULL COMMENT '层级(1,2,3)',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '建立时间',
  
  INDEX idx_parent_id (parent_id),
  INDEX idx_child_id (child_id),
  INDEX idx_level (level),
  UNIQUE KEY uk_parent_child (parent_id, child_id),
  FOREIGN KEY (parent_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (child_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='团队关系表';
```

### 7. 推广收益表 (promotion_earnings)

```sql
CREATE TABLE promotion_earnings (
  id INT PRIMARY KEY AUTO_INCREMENT COMMENT '推广收益ID',
  user_id INT NOT NULL COMMENT '用户ID',
  from_user_id INT NOT NULL COMMENT '来源用户ID',
  level TINYINT NOT NULL COMMENT '层级(1,2,3)',
  amount DECIMAL(8,2) NOT NULL COMMENT '收益金额',
  source_type ENUM('task', 'recharge') NOT NULL COMMENT '来源类型',
  source_id INT COMMENT '来源ID',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  
  INDEX idx_user_id (user_id),
  INDEX idx_from_user_id (from_user_id),
  INDEX idx_level (level),
  INDEX idx_created_at (created_at),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (from_user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='推广收益表';
```

## 索引优化策略

### 1. 主键索引
- 所有表都使用自增主键
- 主键自动创建聚簇索引

### 2. 唯一索引
- 用户手机号、用户名、邀请码
- 用户任务唯一性约束

### 3. 普通索引
- 外键字段
- 常用查询字段
- 排序字段

### 4. 复合索引
```sql
-- 用户任务查询优化
CREATE INDEX idx_user_task_status ON user_tasks(user_id, status);

-- 收益记录查询优化
CREATE INDEX idx_earnings_user_type ON earnings(user_id, type, created_at);

-- 团队关系查询优化
CREATE INDEX idx_team_parent_level ON team_relations(parent_id, level);
```

## 数据字典

### 枚举值说明

#### 用户状态 (users.status)
- `active`: 正常状态
- `inactive`: 未激活
- `banned`: 已封禁

#### 任务分类 (tasks.category)
- `douyin`: 抖音任务
- `kuaishou`: 快手任务
- `sms`: 手工短信任务
- `xiaohongshu`: 小红书任务
- `toutiao`: 今日头条任务

#### 任务状态 (user_tasks.status)
- `pending`: 待开始
- `in_progress`: 进行中
- `completed`: 已完成
- `abandoned`: 已放弃
- `rejected`: 已拒绝

#### 提现方式 (withdrawals.method)
- `alipay`: 支付宝
- `wechat`: 微信
- `bank`: 银行卡

#### 收益类型 (earnings.type)
- `task`: 任务收益
- `promotion`: 推广收益
- `bonus`: 奖励收益
- `withdrawal`: 提现扣减

## 数据备份策略

### 1. 全量备份
- 每日凌晨进行全量备份
- 保留最近30天的备份文件

### 2. 增量备份
- 每小时进行增量备份
- 使用binlog进行增量恢复

### 3. 备份验证
- 定期验证备份文件完整性
- 测试恢复流程

## 性能优化建议

### 1. 查询优化
- 避免SELECT *，只查询需要的字段
- 使用LIMIT限制结果集大小
- 合理使用索引，避免全表扫描

### 2. 分表策略
- 当单表数据量超过1000万时考虑分表
- 按时间或用户ID进行分表

### 3. 缓存策略
- 热点数据使用Redis缓存
- 设置合理的缓存过期时间
- 使用缓存预热策略
